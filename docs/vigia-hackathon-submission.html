VIGIA

Government Infrastructure Intelligence — AI-Powered Citizen Audit Platform

01 — Problem Statement

The infrastructure accountability gap

India's road network spans national highways managed by NHAI, state highways under state PWDs, and rural roads tracked by NRRDA through the PMGSY programme. Despite the scale of annual public expenditure, citizens have no reliable way to verify whether that money reached the road outside their town.

The data exists — NHAI publishes awarded contracts, MoRTH produces annual reports, PMGSY tracks rural roads via the OMMAS portal — but it is scattered across dozens of government portals, locked in unindexed PDFs, and requires domain expertise to interpret.

When a road fails, a citizen must navigate multiple government systems to identify who is responsible, how to file a complaint, and how to submit an RTI. VIGIA collapses this into a single conversational interface.

02 — System Architecture

End-to-end architecture

VIGIA is a Next.js 16 Progressive Web App backed by a stateful LangGraph RAG pipeline running in a Node.js API route. All AI inference runs on Amazon Bedrock. The retrieval layer uses pgvector on RDS as the primary store, with a local SQLite FTS5 database as offline fallback. A dual-track AWS Lambda pipeline ingests government documents daily.

03 — The LangGraph Pipeline

Stateful CRAG orchestration with four graph nodes

The core of VIGIA is a LangGraph StateGraph with four registered nodes (Router, Ingest, Guardrail, UI Hook). After the graph completes, the API route runs LLM synthesis directly on Bedrock with the extracted evidence context. Unlike a simple chain, the graph can loop back on itself — a corrective retrieval (CRAG) mechanism that catches low-confidence retrievals before they reach the user. Each node emits real-time progress steps streamed to the client.

Node 1 — Router

Uses Amazon Nova Lite to classify intent into six categories: condition, complaint, rti, personnel, tender_search, or conversational. Conversational queries short-circuit directly to the LLM, bypassing retrieval entirely. All other intents proceed to the Ingest node.

Node 3 — Guardrail (CRAG)

The most critical node. Implements Corrective RAG with six layered checks:

1
Retrieval grading
Confidence must exceed 0.5 (DATA_VOID_CONFIDENCE_THRESHOLD in guardrail.ts) or evidence is declared a data void.
2
Data void detection
If all evidence falls below threshold, declare a data void rather than hallucinate a response.
3
Query rewrite
Nova Lite rewrites the query from a different angle to widen the retrieval scope.
4
Retry loop
Re-runs the Ingest node with the rewritten query (maximum 2 retries).
5
Authority fallback
On persistent data void, returns exact portal URLs and helpline numbers from the curated Authority Matrix — never invents an answer.
6
Contradiction detection
Cross-checks official documents vs. citizen photo evidence. Flags when NHAI says "maintained" but a photo shows severe damage.

Node 2 — Ingest (Parallel Agents)

Dispatches up to three specialist agents concurrently via Promise.allSettled. Each agent has a 6-second timeout (AGENT_TIMEOUT_MS = 6000) and an independent AbortController. On retry runs, only the Admin agent re-executes with the rewritten query to minimise cost.

Node 4 — UI Hook

A pure computation node — zero LLM tokens consumed. Extracts structured payloads from the evidence state for the frontend: spatial markers (lat/lng for map rendering), budget figures (sanctioned vs. spent for bar charts), citation references, and pending action cards for complaint filing or RTI templates.

Post-graph — LLM Synthesizer

After the graph completes, the API route calls Bedrock directly with the full evidence context injected into the system prompt. Personnel queries automatically upgrade to Nova Pro to reduce contact-detail hallucination. Post-generation, faithfulness scoring runs asynchronously — an LLM-as-judge evaluates whether every factual claim is supported by a retrieved chunk.

04 — Specialist Agent Sub-system

Three agents, one parallel dispatch

The Ingest node dispatches three purpose-built agents, each with a different data modality and reasoning pattern. This lets VIGIA combine documentary evidence, visual ground truth, and physical telemetry into a single corroborated answer.

Admin Agent
ReWOO Plan-and-Execute · Nova Lite
  • Generates a typed retrieval plan — a DAG of up to 4 steps specifying which data source to query and what entities to extract.
  • Executes steps in topological order. Steps without dependencies run in parallel.
  • Downstream steps receive injected results from upstream steps (e.g. district extracted from NHAI to narrow PWD contact search).
  • ReWOO eliminates intermediate reasoning steps, cutting latency vs. standard ReAct agents.
  • Handles international queries via World Bank / OCDS APIs for cross-country comparison.
Vision Agent
Multimodal VLM · Nova Lite (vision)
  • Accepts citizen-uploaded road photos via the chat input (imageUrl in the payload).
  • Passes the image to Nova Lite as a vision language model with an iRAP-standard assessment prompt.
  • Returns structured output: severity (critical/severe/moderate/minor/none), iRAP star rating (1–5), and specific observations.
  • Evidence is tagged as "citizen-claim" trust level with a disclosure that it is unverified.
  • Contradiction detection in the Guardrail cross-checks this against official maintenance records.
Telemetry Agent
GPS + IMU processing · Rule-based (no LLM)
  • Accepts GPS coordinates from the browser geolocation API or extracted from query text via regex.
  • Queries OpenStreetMap (via Overpass / Gati Shakti tool) to identify road type, number, and state at those coordinates.
  • Pipeline integration is complete — accepts IMU anomaly payloads with a 1.5g vertical acceleration threshold.
  • Current build uses prototype anomaly values demonstrating the data model.
  • Foundation for the DePIN telemetry network described in section 07.
05 — Retrieval Architecture

Hybrid vector + keyword search with graceful degradation

VIGIA uses a four-layer retrieval stack designed for resilience. The primary layer is semantic vector search against pgvector on RDS using Amazon Titan Embed v2 (1024 dimensions). Results are merged with a parallel FTS5 keyword search using Reciprocal Rank Fusion, which provably outperforms either approach alone. An optional Cohere cross-encoder reranks the top results for precision.

If the pgvector Lambda proxy is unreachable — due to VPC cold starts or network partitions — the system transparently falls back to a local SQLite FTS5 database. The client never sees an error; it receives slightly lower recall.

Semantic chunking strategy

Government PDFs contain mixed content: prose, financial tables, project lists, and annexures. A naive sentence splitter destroys table structure and loses row–column relationships that carry the actual data (e.g. "Road X | Contractor Y | ₹450 Cr"). VIGIA's PDF parser uses semantic chunking that detects table boundaries and preserves row context within each chunk. Each chunk is embedded independently and stored with metadata: road number, state, district, concessionaire, and source PDF hash for deduplication.

06 — Anti-Hallucination Architecture

Six independent layers against fabrication

Infrastructure data is high-stakes. A wrong phone number, incorrect contract value, or fabricated officer name causes direct harm to citizens filing complaints. VIGIA treats hallucination prevention as a first-class engineering concern.

1
CRAG threshold gating
Evidence below the 0.5 confidence threshold is discarded before reaching the LLM. The model never sees low-confidence chunks to amplify.
2
Authority Matrix fallback
When no reliable evidence exists, VIGIA returns curated portal URLs and helpline numbers from a manually maintained JSON dataset rather than asking the model to guess.
3
COPY-PASTE system prompt
The synthesizer system prompt explicitly instructs the model to reproduce contact details verbatim from evidence — copy-paste, not paraphrase. Deviation is prohibited.
4
Nova Pro upgrade on personnel
Queries involving officer names, phone numbers, and emails are automatically routed to Amazon Nova Pro, which hallucinates contact details at a lower rate.
5
Post-generation faithfulness scoring
After streaming completes, an async LLM-as-judge evaluates whether each factual claim in the response is grounded in the retrieved chunks. Flagged responses are logged.
6
Temporal coherence check
The Guardrail node parses dates in retrieved findings and flags temporal impossibilities — e.g. a maintenance completion date in the future marked as "done".
All names, phone numbers, emails, and financial figures in VIGIA responses must appear verbatim in the retrieved evidence. If a piece of information cannot be traced to a source chunk, the system is explicitly instructed to say so rather than fill the gap.
07 — DePIN Telemetry Network

Crowdsourced road condition intelligence

The single biggest data gap in VIGIA today is live road condition. The NHAI RAMS portal (Pavement Condition Index, International Roughness Index) is credential-gated and not public. CAG audit reports provide sampled IRI data for audited stretches, but coverage is sparse and lagged by years.

The DePIN (Decentralised Physical Infrastructure Network) architecture addresses this with a crowdsourced telemetry layer. Every user's smartphone becomes a road condition sensor. The VIGIA app records the Z-axis accelerometer while driving. Vertical acceleration spikes above the 1.5g threshold (already implemented in the Telemetry Agent) are classified as anomaly events and geo-tagged to the road segment via OpenStreetMap reverse geocoding.

Aggregated across commuters, this produces a real-time road roughness map with coverage that scales with traffic, not government budget.

The Telemetry Agent in the current build already implements this data model end-to-end — the DePIN network is the supply-side infrastructure that replaces prototype values with real crowdsourced data.

08 — Multilingual Voice Interface

13 Indian language locales, script-aware detection

Infrastructure complaints are not an English-speaking problem. A farmer on a PMGSY road in Odisha and a truck driver on NH-44 in Tamil Nadu are equally affected — and equally unable to navigate government portals that assume literacy in a specific language.

VIGIA's voice pipeline accepts audio in any supported language, auto-detects the locale, processes the query in that language, and speaks the response back using Azure Neural Voices. The system prompt is adapted to the detected language so the LLM reasons and responds in the user's language throughout.

Language detection uses a two-pass approach: Unicode script range matching (Devanagari, Tamil script, Malayalam script, etc.) for text inputs, and Azure Cognitive Services automatic language identification for audio across all 13 locales simultaneously.

09 — Offline-First Architecture

Designed for intermittent connectivity

The citizens VIGIA serves are not always on a reliable connection. Rural roads, remote districts, and poor network areas create connectivity gaps. VIGIA's data layer is built offline-first: every message is persisted locally to IndexedDB before a network call is attempted.

The Dexie schema (v2) maintains a full conversation history with sync status per message. When the user goes offline mid-session, their query is queued as "pending" and re-submitted automatically when connectivity returns — without a page reload.

For areas with persistent low connectivity, VIGIA syncs geofenced SQLite tiles from a CDN. Each tile covers a ~40 km geohash cell (precision 4, as coded in lib/edge/sync.ts) and contains road segment data, PWD contact records, and emergency helplines. Queries against this data resolve entirely on-device using sql.js WASM.

12 — Key Innovations

What makes VIGIA different

CRAG on government PDFs
Corrective RAG with a confidence threshold gate is unusual in civic tech. Most RAG systems return whatever they retrieve. VIGIA refuses to answer rather than hallucinate — and has a deterministic fallback (the Authority Matrix) when retrieval fails.
ReWOO plan-and-execute for multi-hop queries
A query like "Who is the executive engineer for the NH-48 stretch in Pune?" requires two retrieval hops: find the district from NHAI data, then look up the PWD officer for that district. ReWOO plans this as a typed DAG before executing, with result injection between steps — no intermediate LLM calls.
Multi-modal evidence corroboration
VIGIA cross-checks evidence from different modalities — documentary (official PDFs), visual (citizen photo, VLM-assessed to iRAP standard), and physical (GPS + IMU telemetry) — and flags contradictions as a first-class finding rather than ignoring them.
Offline-first civic AI
Civic AI tools that assume reliable connectivity exclude the rural populations who need them most. VIGIA's IndexedDB persistence, PWA caching, and geofenced SQLite sync via CDN tiles give degraded but functional access in areas with intermittent or no connectivity.
DePIN for public infrastructure monitoring
Applying decentralised physical infrastructure network principles to government road monitoring is novel. Rather than lobbying for access to NHAI RAMS, VIGIA builds a parallel sensor network from citizen smartphones — with token incentives for data contribution, and a pipeline already built to consume it.
Streaming pipeline trace in the UI
Each graph node emits a real-time progress step streamed to the client: "Classifying intent... Searching 2 sources... Contradiction found — rewriting query... Validating new evidence... Generating response." The user always knows what the system is doing and why it takes time.
