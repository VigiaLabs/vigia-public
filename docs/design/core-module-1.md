# Core Module 1: Repository Architecture & Directory Boundaries

> Strict folder structure and ownership rules for parallel development

---

## Current State

The repository has a flat structure with components scattered across `components/chat`, `components/layout`, `components/ui`. No `lib/` directory exists. All logic is co-located with UI components. No clear separation between frontend presentation, AI orchestration, and data layers.

## Ownership Rules

| Directory | Owner | Rule |
|-----------|-------|------|
| `app/` | Frontend Lead | Routing only. No `fetch()`, no DB calls, no heavy logic. |
| `components/ui/` | Any | Stateless, prop-driven. No imports from `lib/`. |
| `components/chat/` | Frontend Lead | May import from `lib/chat/types.ts` only. |
| `lib/chat/` | AI Integration Lead | Owns streaming, state shape, server actions. |
| `lib/agents/` | Backend/AI Lead | Fenced. No frontend imports allowed. |
| `lib/tools/` | Backend/AI Lead | Pure functions. No React. No DOM. |
| `lib/db/` | Infra Lead | Browser-only. Exports clean async functions. |

## Import Boundary Rules

```
components/ui/     → MAY import: nothing from lib/
components/chat/   → MAY import: lib/chat/types.ts, lib/chat/provider.tsx
components/layout/ → MAY import: nothing from lib/
lib/chat/          → MAY import: lib/agents/, lib/tools/, lib/db/
lib/agents/        → MAY import: lib/tools/
lib/tools/         → MAY import: nothing (leaf module)
lib/db/            → MAY import: nothing (leaf module)
app/               → MAY import: components/*, lib/chat/provider.tsx
```

## Critical Gaps to Fix

1. **No `lib/` directory exists** — Must be created immediately with placeholder `index.ts` files in each subdirectory to establish the structure.
2. **No `types/` directory** — Shared TypeScript interfaces (Message, Source, Citation) are undefined. Team members will create incompatible types without this.
3. **No API route** — `app/api/chat/route.ts` needed for streaming responses.
4. **No `.env.example`** — Team needs to know what environment variables are expected (API keys, model endpoints).
5. **No `eslint` import boundary rules** — Consider adding `eslint-plugin-boundaries` or documenting import rules in CONTRIBUTING.md.

## Implementation Steps

1. Create all `lib/` subdirectories with placeholder `index.ts` exports
2. Create `types/index.ts` with shared interfaces
3. Create `app/api/chat/route.ts` stub
4. Create `.env.example` with placeholder keys
5. Add `CONTRIBUTING.md` documenting ownership and import rules
